# Arche — Coolify Deployment

Deploy Arche on a Coolify-managed VPS with one-click deployment.

## Architecture

```
Coolify Proxy (Traefik)
       │
       ▼ routes to :3000
┌─────────────────────────────────────────────┐
│ Arche Compose Stack                         │
│                                             │
│  ┌─────────┐  ┌────────────────────────┐   │
│  │ postgres│  │ docker-socket-proxy    │   │
│  └─────────┘  └────────────────────────┘   │
│       │                  │                  │
│       ▼                  ▼                  │
│  ┌─────────────────────────────────────┐   │
│  │             web (Next.js)            │   │
│  │  - BFF + spawner                     │   │
│  │  - connects to postgres              │   │
│  │  - spawns OpenCode via socket-proxy  │   │
│  └─────────────────────────────────────┘   │
│                    │                        │
│                    ▼ spawns                 │
│  ┌─────────────────────────────────────┐   │
│  │   OpenCode containers (dynamic)      │   │
│  │   - on arche_internal network        │   │
│  │   - accessed via web app proxy       │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

## Pre-deployment Setup (VPS) — Optional

These steps are optional and only needed for specific features:

### 1. Setup Knowledge Base (optional)

```bash
mkdir -p /opt/arche/kb
# Clone or copy your KB content
git clone <your-kb-repo> /opt/arche/kb
```

### 2. Build custom workspace image (optional)

If using a custom workspace image instead of the default OpenCode image:

```bash
# Clone the repo
git clone https://github.com/peaberry-studio/arche.git /tmp/arche
# Build the workspace image
docker build -t arche-workspace:latest /tmp/arche/infra/workspace-image
```

## Deployment Steps in Coolify

1. **Create new project** in Coolify dashboard

2. **Add Docker Compose resource**
   - Source: GitHub repository
   - Repository: `peaberry-studio/arche`
   - Branch: `main`
   - Docker Compose Location: `infra/coolify/docker-compose.yml`

3. **Configure environment variables** in Coolify UI:

   | Variable | Required | Description |
   |----------|----------|-------------|
   | `ARCHE_DOMAIN` | Yes | Your domain (e.g., `arche.example.com`) |
   | `ARCHE_SESSION_PEPPER` | Yes | Session security key |
   | `ARCHE_ENCRYPTION_KEY` | Yes | Encryption key for secrets |
   | `ARCHE_INTERNAL_TOKEN` | Yes | Internal API token |
   | `ARCHE_SEED_ADMIN_EMAIL` | Yes | Initial admin email |
   | `ARCHE_SEED_ADMIN_PASSWORD` | Yes | Initial admin password |
   | `ARCHE_SEED_ADMIN_SLUG` | No | Admin URL slug (default: `admin`) |
   | `OPENCODE_IMAGE` | No | Workspace image (default: `ghcr.io/anomalyco/opencode:1.1.45`) |
   | `KB_HOST_PATH` | No | Knowledge Base path on host |
   | `ARCHE_COOKIE_SECURE` | No | Cookie security (default: `true`) |

   **Note:** `SERVICE_PASSWORD_POSTGRES` is automatically generated by Coolify.

4. **Generate secrets** (run on your local machine):
   ```bash
   echo "ARCHE_SESSION_PEPPER=$(openssl rand -base64 32)"
   echo "ARCHE_ENCRYPTION_KEY=$(openssl rand -base64 32)"
   echo "ARCHE_INTERNAL_TOKEN=$(openssl rand -base64 32)"
   ```

5. **Set domain** for the `web` service in Coolify's domain settings

6. **Deploy** the stack

## Verification

After deployment:

1. **Check services are running:**
   ```bash
   docker compose -p arche ps
   ```

2. **Verify web app responds:**
   ```bash
   curl https://<your-domain>/
   ```

3. **Check database connection:** Log in to the admin panel

4. **Test container spawning:** Create a new workspace

5. **Verify healthchecks:**
   ```bash
   docker compose -p arche ps
   # All services should show "healthy" or "running"
   ```

## Troubleshooting

### Database connection errors

Check postgres is healthy:
```bash
docker compose -p arche logs postgres
```

### Migration issues

Re-run migrations manually:
```bash
docker compose -p arche run --rm migrate
```

### GHCR authentication (private images)

If using private images from GHCR, configure registry credentials in Coolify's server settings.

## Backups

Configure postgres backups via:
- Coolify's backup features
- External backup solution mounting the `postgres_data` volume
- pg_dump cron job

## Updates

1. Push changes to the GitHub repository
2. In Coolify, click "Redeploy" on the resource
3. Coolify will rebuild and restart the stack

## Notes

- **Workspace routing:** Users access OpenCode workspaces via the web app's proxy. No wildcard DNS or additional Coolify configuration needed.
- **Traefik:** This compose does NOT include Traefik — Coolify provides its own reverse proxy.
- **Network:** The `arche_internal` network is used for communication between the web app and spawned OpenCode containers.
