# Soporte - Procedimientos

## Crear una entrada de KB a partir de un ticket

1. Crear una nota nueva usando [[Templates/Soporte - KB Entry]].
2. Escribir el “Problema” con el síntoma real (sin suposiciones).
3. En “Diagnóstico”, separar hipótesis de evidencia.
4. En “Solución”, incluir verificación concreta (qué debe observar el usuario).
5. Enlazar la nota desde `Company/Product/01 - Soporte - Indice KB.md`.

## Cuándo crear nota nueva vs editar una existente

- Crear nota nueva si:
  - Cambia el tipo de problema, causa o solución.
  - La nota existente se está volviendo larga o mezcla casos.
- Editar nota existente si:
  - Es el mismo caso y solo añades una verificación, un matiz o un ejemplo.

## Criterios de calidad (antes de “cerrar”)

- Título describe el síntoma o el objetivo.
- Solución tiene pasos verificables.
- No hay datos sensibles (tokens, cookies, credenciales).
- Está enlazada desde el índice de soporte.
